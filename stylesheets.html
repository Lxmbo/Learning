Cascading Style Sheets!

Now that we've covered a vast majority of html we will now be moving onto Cascading Style Sheets or CSS for short.

What is CSS?

CSS is a language, not really a programming language but a language non the less. It allows us to make changes to HTML! CSS is a core component of most websites. Allowing us to move and color different things.

The typical syntax for CSS or RULES look something like this:

selector {
property: value;
}

Several examples of this are
h1{ color:purple; } this allows us i make all h1s in our code the color purple.
img{ width:200px; height:100px; }while this allows us to change the size of an image.
etc.

Working now with text align. This doesnt not allow us to align the text on the page but rather it aligns it within the element. The h1, and h2 of this page are aligned to center. The only reason they are in the center of the page is because the element that they are confined in goes all the way across the page.

Font weight: font weight is the "boldness of the font" h1 comes with a already bolded font, and some fonts dont have access to some font weights, usually extra bold or extra thin. We have changed the font weight of the h1 on this page to normal(400).

Text decoration: The appearence of certain text, underline overline, like through, etc. the h1 and h2 on this page have a text decoration of white underline.

Something else that can be of good use is line height attribute. Allowing us to make things double spaced. like all the paragraphs on this page. Also the h1 in this page has a letter spacing of 15px.

Fonts and stuff!

Font-size: used to change the size of fonts. Kinda tricky becausse we have so many different units that measure the actual size of the fonts.
Pixels(aka) PX: most used absolute unit. 1 px does not necessarily equal the width of exactly one pixel on your machine! (not recommended for responsive websites).

Font family: This is how we change fonts. Now we cannot use every font, we have to use browser safe fonts, until we learn how to import our own fonts into webpages. Most of the time you will not see one font in code but rather a FONT STACK, or a list of fonts we would like to use in order of how we would like to have them used.

Selectors!

Universal element selectors: (*) selects every single element.
Element selectors: (button, h1, h2, p, etc.)

You can use a comma to select multiple things at once and give them the same styles.

ID selector: you can add an ID on to a single element and provide a hook, that way we can select it in CSS and give it an exact stlye without ruining the style of anything else.

Class selector: Similar to Id, a class can be applied to multiple elements. allowing us to group them together and style them similarly. To use the class selector add class="(classnamehere)" then in CSS to start styling we select the class as follows .classname{ color: x ;}

Descendant selector: li a {color:gold}; Written with a space the Descendant selector allows us to select specific tags within a different tag. In this example we're selecting all anchor (a) tags that have been nested within the list (li) tag.

Adjacent selector: h1+p{color; red}; Adjacent selectors select a tag that is immdeiately proceeded by another tag. In this example we are selecting only the paragraphs (p) that are proceeded by a h1 heading tag on the same level. All other paragraphs besides the ones that fit this criteria will remain unchanged. This is NOT nesting.

Direct child selector:div>li{color:white;} The direct child selector selects only the tag that is one level down from the parent tag.

Attribute selector:input[type="text"] width: 2em; color:white; allows us to select elements based on some particular attribute. In this particular case we are selecting all inputs where "type" is set to text. The type could be a lot of things: password, email search, etc. Class, type, and Id are all attributes. So thigns like div[class="123"]{color:red;} are possible.

Pseudo classes: keyword added to a selector that specifies a special state of selected elements.

  1. :active
    2. - when button or link is clicked it will change the style based on set parameters.
  2. :checked
    3. - when a checkbox or radio button is checked style will changed based on set parameters
  3. :first
  4. :first-child
  5. :hover
    6. - when mouse hovers over element changes style based on parameters.
  6. :not()
  7. :nth-child
  8. :nth-of-type

Pseudo elements: just about the same as pseudo classes, but can go a little deeper into the actual code selecting very specific things like single letters in a word for example.

  1. ::after
  2. ::before
  3. ::cue
  4. ::cue-region
  5. ::first-letter
    6. - Selects the first letter of a given element ie h2::first-letter{color:red;}
  6. ::first-line
  7. ::selection
  8. ::slotted()

The CSS Cascade:

The order out styles are encountered actually matters. The browser reads from top to bottom and selects the most recent style for a particular element that it has seen. This is also the same if we have multiple style sheets linked to one given page. The most recently read style sheet will be the one the browser uses.

Specificity: How the browser decides which rules to apple when multiple rules could apply to the same element.
It is a measure of how specific a given selector is. The more specific selector "wins".
For example an #id selector is extremely specific and will beat out a class and element selector. A class selector is kinda specific and will beat out an element selector, but lose to an #id. Lastly and element selector isn't very specific, and will generally only be used if there is no competition. If a color has a conflict but a font stlye does not, the color will be subject to specificity. The font style however will not and will go all the way down and be used. Even if it is only under an element selector. Because that specific style doesnt have anything to compete with.
Inline styles are the most specific and can not be overwritten in a style sheet. So avoid inline styles. The same with !important (which is used after a style). It signals to the broswer that this should be the last stop. The only thing considered. That's why it is not recommended. Don't use these!

Box Model:

Box model is the idea that everything in CSS has a box around it. Wether it be a picture, some text, or a search bar. There's a box around everything. We can change those box's size with height and width. These two elements allow us to make changes to the actual area where the content is placed. By default the width of a box will go all the way across the page. If this is changed it will alter something like (text-align:center). This is because if the width box is changed, the content which in this case is the text will go with it. To change this we use margin.

The Box model is simple, but has a lot of content which I will not go over here. Please proceed to the MDN link provided for Box Model.
